## 1. Pipeline Stages in Order

### Canonical workflow order (from `workflow.md` `stage_order`)

```
intake → segmentation → seed-review → landmarks-orientation → roi → measurement
```

`workflows/planning.py` defines stage field requirements for workflow YAML extraction but does **not** include `intake` or `seed-review` in `STAGE_FIELDS` — those are operational stages, not workflow-schema sections.

---

### Stage 1: Intake

| Field | Value |
|-------|-------|
| **Driver** | `stages/intake.py` → `run_intake(dicom_path, output_dir)` |
| **Inputs** | DICOM directory path |
| **Processing** | `processing/dicom.load_dicom` → histogram/threshold calibration via `processing/calibration` |
| **Outputs** | See `domain/artifact_contracts.IntakeArtifacts` |

**Artifacts:**
- `intake/volume_metadata.json` — spacing, affine, scanner profile, histogram, derived thresholds, fingerprint, provenance
- `intake/orientation_report.md` — human-readable load summary
- `intake/stage_report.json` — confidence, evidence, flags, recommended_action

**Review-worthy evidence:**
- Scanner detection flags (`unknown-scanner-profile`)
- Segmentation threshold bimodality status (`segmentation_threshold_analysis.status`: analyzed / not-bimodal / derived / failed)
- `threshold_flags` list
- **No volume emitted** — resampling deferred to segmentation (line 104 of `intake.py`)
- Screenshot slot: `intake/screenshot_001.png` (path declared; not auto-generated by driver)

**Confidence:** `high` if no flags; `medium` + `flag` action if flags present.

---

### Stage 2: Segmentation

| Field | Value |
|-------|-------|
| **Driver** | `stages/segmentation.py` → `run_segmentation(...)` |
| **Inputs** | Either (a) `dicom_path` / `intake_metadata_path` for full pipeline, or (b) in-memory `volume` + `spacing` + `workflow_thresholds` (legacy workbench path) |
| **Processing chain** | DICOM load → isotropic resample (`processing/resample`) → median filter (`processing/preprocess`) → dual threshold mask+markers (`processing/threshold`) → component extraction + FOV prefilter (`processing/segmentation`) → seed assignment (heuristic or `seeds.json`) → marker watershed (`processing/watershed`) → prune to seed CC → sanity checks (`processing/sanity`) |
| **Status values** | `ready`, `needs-seeds`, `failed` |

**Artifacts** (`domain/artifact_contracts.SegmentationArtifacts`):
- `segmentation/labels.nii.gz` — per-bone label volume
- `segmentation/filtered.nii.gz` — median-filtered intensity
- `segmentation/structure_assignments.json` — `{status, assignments, component_assignments, flags}`
- `segmentation/seeds.json` — scan fingerprint + component→bone proposals
- `segmentation/metadata.json` — full provenance, per-bone stats, component summary, pruning stats
- `segmentation/stage_report.json`
- `segmentation/masks/{femur,tibia,...}.nii.gz` — per-bone binary masks (full pipeline only)
- `segmentation/components.nii.gz` — **only** when `needs-seeds` (ambiguous bone identity)

**Review-worthy evidence:**
- Structure assignment map (which label = which bone)
- Escalation flags: `histogram-not-bimodal`, `ambiguous-bone-identity`, `articular-bridging-suspected`, `unknown-scanner`, `anisotropic-resampled`, confounder strings, sanity warnings
- Component geometry (centroids, bbox, edge-face touches) in metadata
- Threshold observations vs workflow expectations (`compare_thresholds`)
- `render_qc` parameter accepted but **explicitly no-op** (`del render_qc` at line 312)
- Screenshot slot: `segmentation/screenshot_001.png`

**Confidence:** `high` / `medium` / `low` via `confidence_for_segmentation()`; `needs-seeds` → `low` + pause.

### Stage 2b: Seed Review (workflow stage, not a Python stage driver)

| Field | Value |
|-------|-------|
| **Trigger** | Segmentation returns `status: needs-seeds` |
| **Mechanism** | Interactive PyVista scene + durable event log (`workflows/review.py`) |
| **State** | `domain/seed_curation.SeedState` + `domain/review_events.translate_events()` |
| **Re-run** | Pass curated `seeds_path` to `run_segmentation()` → `ambiguity-resolved-via-seeds` flag |

**Review surface:** Pick components in 3D; palette keys 1–5 assign femur/tibia/patella/fibula/unassigned. Events translated to `SeedAssignmentOp` list.

**Artifacts:** Updated `seeds.json` written back; segmentation re-invoked.

### Stage 3: Landmarks & Orientation

**Two parallel paths exist — important for LabTask:**

#### Path A: Automated (`stages/landmarks_orientation.py`)

| Field | Value |
|-------|-------|
| **Driver** | `run_landmarks_orientation(segmentation_artifacts, workflow_landmarks, workflow_orientation, output_dir)` |
| **Status** | **Legacy / semi-retired** — PCA orientation retired (lines 45–46, 80–81) |
| **Landmark methods** | Femoral: surface mesh features (`processing/surface`). Tibial: slice scans (articular area threshold, growth plate bone-fill-ratio drop) |
| **Orientation** | `compute_orientation_frame()` from landmark pairs; `orientation_applied: False` |

#### Path B: Visual / Agent-driven (`stages/visual_landmarks.py`) — **active path per code comments**

| Field | Value |
|-------|-------|
| **Driver** | Agent places landmarks; pipeline calls `emit_positions(placed_landmarks, ...)` |
| **Support** | `processing/rendering.prepare_landmark_session()` — dual meshes + KDTrees |
| **Snapping** | `processing/snapping.snap_to_surface()` (femoral 3D), `snap_to_slice()` (tibial 2D) |
| **Validation** | `processing/backstop.compute_backstop()` — per-landmark signal ensemble with `feedback` |
| **Frame** | `processing/femoral_frame.build_femoral_frame()` → `positions._derived_frame` |

**Shared outputs:**
- `landmarks/positions.json` — landmark list with voxel/physical coords, per-landmark confidence, evidence
- `landmarks/orientation_frame.json` — axes, rotation_matrix, explanation
- `landmarks/transform_matrix.json` — PCA stub (identity, not applied)
- `landmarks/oriented_labels.npy` — copy of labels (no actual reorientation)

**Review-worthy evidence:**
- Per-landmark `confidence` and `requires_user_confirmation`
- IIOC plausibility: articular↔growth_plate interval must be 1–100 slices (`visual_landmarks._check_iioc_interval`, `landmarks_orientation._tibial_interval_issue`)
- Backstop `signals` dict per landmark
- `rendering.validate_segmentation_for_landmarking()` pre-flight
- Static renders: `rendering.render_surface_view()`, `rendering.render_slice_view()`
- Interactive: `workflows/review.landmark_picking_code()` — PyVistaTrameHelper

**Stage confidence aggregation (visual):** ANY low → stage low; ANY medium or IIOC issue → medium; all high → high.

## 3. Workflow Composition

```
DICOM → intake → segmentation → [seed-review] → landmarks → roi → measurement
```

Workflow contract (`workflows/loading.py`): requires `workflow_id`, `thresholds`, `landmarks`, `roi_definitions`, `measurements`, `orientation_protocol`, `acceptance_checks`, `sources`.

### Mouse-knee OA core measurements

**Femoral (3D surface):** `distal_femoral_length`, `distal_femoral_width`, `distal_femoral_ratio`

**Tibial (2D slice):** `tibial_iioc_height` (boundary_slice_count × 0.0105 mm), `tibial_width`, `tibial_iioc_ratio`

**Trabecular (ROI):** BV/TV, Tb.Th, Tb.N, Tb.Sp per compartment

### Processing module roles

| Module | Role |
|--------|------|
| `dicom.py` | DICOM→ScanVolume, fingerprint |
| `threshold.py` | Liberal mask + strict opened markers |
| `segmentation.py` | Components, seed assign, marker labeling |
| `watershed.py` | Grow markers, prune disconnected CCs |
| `rendering.py` | Pre-flight, session prep, PNG renders |
| `backstop.py` | Landmark validation with retry feedback |
| `snapping.py` | Pick→coordinate (3D surface, 2D slice) |
| `femoral_frame.py` | ML/AP/SI frame from condyle landmarks |

## 4. Feedback / Correction Pattern

### `workflows/feedback.py`

Pattern-matches natural language to `DomainOperation` types:

| User signal | Operations |
|-------------|------------|
| Wrong bone | `reassign_component` or `inspect_region` |
| Missing/gap/hole | `adjust_threshold` (lower) + `inspect_region` |
| Noise/speckles | `adjust_threshold` (raise) |
| Too small/clipped | `adjust_boundary` (expand) |
| Too big/bleeding | `adjust_boundary` (shrink) |
| Vague pointer | `inspect_region` |
| Default | `inspect_current_stage` |

**"Growth plate is 10 slices off":** No dedicated operation. Falls through to `inspect_region`. Correction path:

1. Human flags `growth_plate_proximal` (visual review or backstop rejection)
2. Agent re-picks on correct slice via `snap_to_slice(pick_slice_index, ...)` — user selects slice ~10 indices away
3. `emit_positions()` with updated landmarks
4. `aggregate_confidence()` re-checks IIOC [1, 100] slices
5. Re-run ROI + measurement from updated `positions.json`

### `workflows/review.py`

jupyter-workbench kernel exec: PyVista scenes, `DurableEventLog` polling, `capture_screenshot`.

### `workflows/explain.py`

Explain-before-execute for threshold, reassignment, landmark decisions.

### Skill agentic loop (`reference-calibration.md`)

```
RENDER → REASON → WRITE detection code → VALIDATE vs ground_truth.json → ITERATE → human confirm
```

## 5. Ground Truth Gates

### `ground_truth.json` (skill + voxbone assets — identical)

| Key | Gate / Threshold |
|-----|------------------|
| `distal_femur_length_mm` | gate [2.0, 2.7] mm |
| `distal_femur_width_mm` | gate [2.3, 4.2] mm |
| `femur_width_length_ratio` | normal_max 1.28, oa_min 1.30, gate [1.0, 1.8] |
| `tibial_width_mm` | gate [2.2, 3.8] mm |
| `tibial_IIOC_max_height_mm` | gate [0.5, 1.2] mm |
| `tibial_IIOC_height_width_ratio` | cutoff 0.28 (>normal, <OA), gate [0.15, 0.40] |
| `compartment_height_mm` | gate [0.3, 1.1] mm — asymmetry flags bad GP placement |
| `growth_plate_thickness_mm` | gate [0.1, 0.35] mm |
| `voxel_size_um` | 10.5; Scanco thresholds 220/320/270 (unitless) |

mm gates are figure-read sanity limits; ratio thresholds are verbatim from paper.

### Workflow `acceptance_checks` (mouse-knee fixture)

- Segmentation: bone volume ordering, condyle separation
- Landmarks: femoral length 2.0–2.6 mm, groove-notch anatomy
- ROI: growth plate visible, IIOC 50–100 slices
- Measurement: ratio sanity, exact slice multiple, tibial ratio 0.15–0.45

### Contradictions (docs vs code)

1. **IIOC slice range:** stage drivers use 1–100; acceptance_checks/backstop use 50–100
2. **Orientation:** workflow `requires_orientation: true` for tibial landmarks; runtime `orientation_applied: False` (PCA retired, no volume reorientation)

| Gap | Detail |
